# Render Deployment Guide

## Prerequisites
- GitHub repository with your code
- Render.com account
- MongoDB connection string (MONGO_URL)
- Environment variables configured

## Deployment Steps

### 1. Push to GitHub
```bash
git init
git add .
git commit -m "Initial commit for Render deployment"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

### 2. Connect GitHub to Render
1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click "New +" and select "Blueprint"
3. Connect your GitHub repository
4. Authorize Render to access your repositories

### 3. Configure Environment Variables
Before deploying, set these environment variables in Render dashboard:

**Backend Environment Variables:**
- `MONGO_URL` - Your MongoDB connection string
- `DB_NAME` - Database name
- `EMERGENT_LLM_KEY` - Your Emergent LLM API key
- `BACKEND_URL` - Your backend service URL (will be auto-generated by Render)

**Frontend Environment Variables:**
- `REACT_APP_API_URL` - Backend API URL (e.g., https://urdu-backend.onrender.com)

### 4. Deploy
1. Select the repository
2. Render will automatically detect `render.yaml`
3. Review the services (backend and frontend)
4. Click "Deploy"

### 5. Monitor Deployment
- Backend will be available at: `https://urdu-backend.onrender.com`
- Frontend will be available at: `https://urdu-frontend.onrender.com`
- Check logs in the Render dashboard for any issues

## Environment Variables Setup

Create `.env` files locally (do NOT commit):

**backend/.env:**
```
MONGO_URL=mongodb+srv://user:password@cluster.mongodb.net/?retryWrites=true&w=majority
DB_NAME=urdu
EMERGENT_LLM_KEY=your_key_here
```

**frontend/.env:**
```
REACT_APP_API_URL=http://localhost:8000
```

For production (Render), use the Render dashboard to set:
- Backend: `REACT_APP_API_URL=https://urdu-backend.onrender.com`

## Important Notes

- Free tier on Render spins down after 15 minutes of inactivity
- For production, consider upgrading to paid plans
- The `serve` package will be installed automatically for frontend static hosting
- Make sure MongoDB allows connections from Render IPs (whitelist 0.0.0.0/0 or specific Render regions)

## Troubleshooting

### Backend not starting
- Check `MONGO_URL` is correct
- Verify all environment variables are set
- Check logs in Render dashboard

### Frontend can't connect to backend
- Ensure `REACT_APP_API_URL` is set correctly
- Check CORS settings in backend
- Verify backend is running

### Build failing
- Check `npm install` output for dependency issues
- Verify `package-lock.json` exists
- Ensure Node version compatibility
